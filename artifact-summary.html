Local Artifact 功能与实现总结

一个将对话中的工作"捕获"为单文件、可在浏览器直接打开的交互式页面 —— Claude Code artifact 功能的本地、离线等价实现。无需云端托管、无需后端、无需构建步骤。

什么是 Artifact

官方定义："an artifact is a capture of work, not an application" —— 它是对一份工作的"快照捕获"，而不是一个真正的应用。它没有后端、没有跨刷新的持久化、表单提交不会发送到任何地方。

一切（标记、样式、脚本、数据）都内嵌在一个文件里，用户双击就能从 file:// 打开。这正是 Claude Code 第 25 周（2026-06-15）推出的 artifacts 功能的本地、离线版本。

❌ 不做什么（约束）

  • · 不调用任何远程 API
  • · 不依赖服务器 / 后端状态
  • · 不拆分成多个 CSS/JS 文件
  • · 渲染体积不超过 16 MiB

✓ 做什么（能力）

  • · 单文件自包含、双击即开
  • · 数据硬编码为顶部 JSON
  • · 深色/浅色双主题
  • · 就地更新 + 版本管理
